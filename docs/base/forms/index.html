These are the base styles for creating forms.

Dependencies

  • Sass - Either Libsass or Ruby Sass.
  • NPM or Bower - To install and manage the Scales repositories. Not technically required as you could manually install.

Installation


  npm install --save-dev scales-base-forms

or


  bower install --save-dev scales-base-forms

Usage

Scales provides default stacked forms, forms using an unordered/ordered list, inline forms, and inline form containers with stacked label/input. It also provides a few helper classes for additional fields within the form.

Default stacked

The default doesn't require any special classes.


  <form>
      <label for="account">Account #<small class="form-label--additional">(Cannot Change)</small></label>
      <input class="text-input" id="account" type="num" value="123456789" readonly>

      <label for="email">Email</label>
      <input class="text-input" id="email" type="email" placeholder="Email">
      <span class="helper-text">Enter your email address</span>

      <label for="password">Password</label>
      <input class="text-input" id="password" type="password" placeholder="Password" disabled>

      <label for="gender">Gender</label>
      <select id="gender">
          <option>Female</option>
          <option>Male</option>
          <option>Other</option>
      </select>

      <label>Introduce yourself</label>
      <textarea></textarea>

      <label for="newsletter">
          <input id="newsletter" type="checkbox"> Send me the newsletter
      </label>

      <button type="submit" class="btn">Sign in</button>
  </form>

Using a list

Use .form-list on a ul inside the form and .input-container on the li's.

To see the benefits of this approach, read Using Unordered Lists for Forms.


  <form>
      <ul class="form-list">
          <li class="input-container">
              <label for="account">Account #<small class="form-label--additional">(Cannot Change)</small></label>
              <input class="text-input" id="account" type="num" value="123456789" readonly>
          </li>
          <li class="input-container">
              <label for="email">Email</label>
              <input class="text-input" id="email" type="email" placeholder="Email">
          </li>
          <li class="input-container">
              <label for="password">Password</label>
              <input class="text-input" id="password" type="password" placeholder="Password" disabled>
          </li>
          <li class="input-container">
              <label for="gender">Gender</label>
              <select id="gender">
                  <option>Female</option>
                  <option>Male</option>
                  <option>Other</option>
              </select>
          </li>
          <li class="input-container">
              <label>Introduce yourself</label>
              <textarea></textarea>
          </li>
          <li class="input-container">
              <label for="newsletter">
                  <input id="newsletter" type="checkbox"> Send me the newsletter
              </label>
          </li>
      </ul>
      <button type="submit" class="btn">Sign in</button>
  </form>

Inline

Use .form-inline to force the form elements into a single inline row.


  <form class="form-inline">
      <label for="email">Email</label>
      <input class="text-input" id="email" type="email" placeholder="Email">

      <label for="password">Password</label>
      <input class="text-input" id="password" type="password" placeholder="Password" disabled>

      <button type="submit" class="btn">Sign in</button>
  </form>

Demo

Inline input containers with stacked label/input

Use .input-container--inline with .input-container to force the containers into a single row and control the vertical alignment of the fields.

KNOWN ISSUE: This works really nicely for label/input pairs, but for elements like button that would usually be alone, the vertical alignment (default for Scales is top) will probably look weird and you will have to write additional styles to control this problem.


  <form>
      <ul class="form-list">
          <li class="input-container input-container--inline">
              <label for="email">Email</label>
              <input class="text-input" id="email" type="email" placeholder="Email">
          </li>
          <li class="input-container input-container--inline">
              <label for="password">Password</label>
              <input class="text-input" id="password" type="password" placeholder="Password" disabled>
          </li>
          <li class="input-container input-container--inline">
              <button type="submit" class="btn">Sign in</button>
          </li>
      </ul>
  </form>

Helper Classes

Scales comes with a few classes that can be applied to elements within a form to add additional functionality

Form Label

This class can be used to make an element look like a label if an actual label isn't suitable.

.form-label

Form Label Additonal

This class is for extra help text in a label.

.form-label--additional

Use like this:


  <label for="account">Account #<small class="form-label--additional">(Cannot Change)</small></label>
Demo

Text Input

Instead of a [type] selector for each kind of form input, just use this class to target all of the inputs that are meant for text.

.text-input

Use like this:


  <input type="text" class="text-input">
  <input type="email" class="text-input">
  <input type="password" class="text-input">

Helper Text

This class displays extra help text after a field when that field is in focus.

NOTE: Can't be used with .form-inline.

.helper-text

Use like this:


  <label for="email">Email</label>
  <input class="text-input" id="email" type="email" placeholder="Email">
  <span class="helper-text">Enter your email address</span>
