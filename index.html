Scales Principles

Purposely Design Free

Scales is purposely design free, so you won't have to undo anything to implement your own design. You bring the design, Scales lays the foundation to build it on.

100% Responsive and Layout Independent

Scales is not interested in layout. All of the patterns and native elements in Scales are flexible so they adapt to whatever container you decide to put them in.

Make Your Own Decisions

Scales makes very few assumptions and let's you choose the tools you want to use. Grid frameworks, responsive breakpoints, build tools, design...these are all things you should decide to use for yourself. A CSS framework should help me solve design problems. It shouldn't do my work for me.

Replace and Extend Styles, Don't Override.

Scales is built on the Open/Closed Principle. All of the default values in Scales can be replaced using Sass variables and should be extended using new classes rather than causing unintended consequences by modifying the base patterns.

Scales Features

Sass

Scales is built using Sass. You can always use the minified output if you don't use Sass, but you lose out on the ability to configure Scales.

BEM Naming Convention

Scales uses BEM style naming convention for the OOCSS Patterns and Base Elements. You don't have to use BEM in your own code, but give it a try, you might like it!

Composable OOCSS Patterns

The OOCSS patterns that come with Scales are highly reusable and easy to combine with each other and the components of your design. Each pattern does one thing well without side effects.

Base Elements

Scales covers the basic HTML elements to give your design a solid starting point. Scales handles the inconsistencies between browsers so you never have to think about them.

Immutable Overrides & Utilities

Scales comes with Normalize, a companion Reset file, and a tiny Clearfix utility. It also comes with overrides for spacing and element size.

Highly Configurable

Everything in Scales is configurable. Default values for every pattern, utility, base element, and override can be replaced using Sass variables. This allows you to use your design right from the start.
